df DataFrames

A DataFrame is a 2-D, labelled, heterogeneous table — the core Pandas data structure. Think of it as an ordered dict of Series that all share the same index.

Creating a DataFrame
import pandas as pd
import numpy as np

# From a dict of lists
df = pd.DataFrame({
    'name':  ['Alice', 'Bob', 'Carol'],
    'score': [91, 78, 85],
    'grade': ['A', 'B', 'B']
})

# From a NumPy array — specify column names
df2 = pd.DataFrame(np.random.rand(3, 3), columns=['x', 'y', 'z'])

# From a CSV / Excel file
df3 = pd.read_csv('data.csv')
df4 = pd.read_excel('data.xlsx')
Inspecting a DataFrame
df.head(n)

First n rows (default 5).

df.tail(n)

Last n rows.

df.info()

Column types, non-null counts, memory.

df.describe()

Summary statistics for numeric columns.

df.shape

Tuple (rows, cols).

df.dtypes

Data type of every column.

Selecting Data
# Single column → Series
df['score']
df.score           # dot notation (no spaces/special chars)

# Multiple columns → DataFrame
df[['name', 'score']]

# Row selection by label  — .loc[row, col]
df.loc[0]                          # row 0
df.loc[0:1, ['name', 'score']]    # rows 0–1, two cols

# Row selection by position — .iloc[row, col]
df.iloc[0]                          # first row
df.iloc[:2, 1:]                    # rows 0-1, col 1 onwards

# Boolean filtering
df[df['score'] > 80]
df[(df['score'] > 70) & (df['grade'] == 'B')]
df.query("score > 80 and grade == 'A'")
Modifying Data
# Add / overwrite column
df['pass'] = df['score'] >= 80

# Apply a function element-wise
df['score_norm'] = df['score'].apply(lambda x: (x - 50) / 50)

# Rename columns
df.rename(columns={'score': 'marks'}, inplace=True)

# Drop rows / columns
df.drop(index=0)                     # drop row 0
df.drop(columns=['pass'])             # drop column

# Set a column as index
df.set_index('name', inplace=True)
df.reset_index(inplace=True)        # restore default integer index
Tip: Prefer df.loc[row, col] for label-based access and df.iloc[row, col] for positional. Avoid chained indexing like df['col'][0] — it can silently fail on assignment (SettingWithCopyWarning).

∑ groupby

groupby follows the Split → Apply → Combine pattern: split the data into groups, apply a function to each group independently, then combine the results into a new structure.

Basic Aggregation
sales = pd.DataFrame({
    'region': ['North', 'South', 'North', 'South', 'North'],
    'product': ['A', 'A', 'B', 'B', 'A'],
    'revenue': [200, 150, 300, 250, 180],
    'units':   [20, 15, 30, 25, 18]
})

# Group by one key → aggregate
sales.groupby('region')['revenue'].sum()
sales.groupby('region').mean(numeric_only=True)

# Group by multiple keys
sales.groupby(['region', 'product']).sum()

# Multiple aggregations at once with .agg()
sales.groupby('region').agg(
    total_rev   = ('revenue', 'sum'),
    avg_units   = ('units',   'mean'),
    max_revenue = ('revenue', 'max')
)
Transform — broadcast result back to original shape
# Normalise each row by its group total
sales['rev_pct'] = (
    sales['revenue'] /
    sales.groupby('region')['revenue'].transform('sum')
)

# Z-score within each region
sales['rev_z'] = sales.groupby('region')['revenue'].transform(
    lambda x: (x - x.mean()) / x.std()
)
Filter — keep only groups matching a condition
# Keep only regions with total revenue > 500
sales.groupby('region').filter(lambda g: g['revenue'].sum() > 500)
Apply — arbitrary per-group function
# Return top-1 row per region (custom function)
def top_row(g):
    return g.nlargest(1, 'revenue')

sales.groupby('region').apply(top_row, include_groups=False)
Common Aggregation Functions
  • sum() · mean() · median() · std() · var()
  • min() · max() · count() · nunique()
  • first() · last() · prod()
  • agg(['sum', 'mean']) — multiple functions at once
Note: By default the group keys become the index. Pass as_index=False to groupby() to keep them as regular columns — useful before a merge.

⋈ merge / join

Pandas offers three combining functions. Use pd.merge() for SQL-style joins on key columns; use df.join() as a shortcut for index-based joins; use pd.concat() to stack DataFrames by axis.

pd.merge() — SQL-style joins
employees = pd.DataFrame({
    'emp_id':  [1, 2, 3, 4],
    'name':    ['Alice', 'Bob', 'Carol', 'Dave'],
    'dept_id': [10, 20, 10, 30]
})

departments = pd.DataFrame({
    'dept_id': [10, 20],
    'dept':    ['Engineering', 'Marketing']
})

# INNER join (default) — only matching rows
pd.merge(employees, departments, on='dept_id')

# LEFT join  — all employees, NaN if no dept match
pd.merge(employees, departments, on='dept_id', how='left')

# RIGHT / OUTER joins
pd.merge(employees, departments, on='dept_id', how='right')
pd.merge(employees, departments, on='dept_id', how='outer')

# Different column names in each table
pd.merge(df_a, df_b, left_on='user_id', right_on='id')

# Join on index
pd.merge(df_a, df_b, left_index=True, right_index=True)

# Suffix for overlapping column names
pd.merge(df_a, df_b, on='id', suffixes=('_left', '_right'))
Join Types — Visual Summary
INNER (default)

Rows that exist in both tables. No NaNs from the join keys.

LEFT

All rows from the left table; NaN where no right match.

RIGHT

All rows from the right table; NaN where no left match.

OUTER (full)

All rows from both tables; NaN wherever no match exists.

df.join() — index-based shortcut
# Both DataFrames must share a common index
a = pd.DataFrame({'A': [1, 2]}, index=['x', 'y'])
b = pd.DataFrame({'B': [3, 4]}, index=['x', 'y'])

a.join(b)             # left join by default
a.join(b, how='inner')
pd.concat() — stacking DataFrames
# Stack rows (axis=0, default)
combined = pd.concat([df1, df2], ignore_index=True)

# Stack columns side by side (axis=1)
combined = pd.concat([df1, df2], axis=1)

# Retain source labels in a multi-level index
pd.concat([df1, df2], keys=['2023', '2024'])
merge vs concat: Use merge when combining on a shared key/column. Use concat when simply stacking tables that share the same schema (e.g., monthly exports).

⊞ Reshape

Reshaping means changing the layout of a DataFrame — converting between wide and long formats, creating pivot tables, and working with multi-level (hierarchical) indices.

pivot_table — spreadsheet-style summaries
df = pd.DataFrame({
    'region':  ['North', 'North', 'South', 'South'],
    'product': ['A', 'B', 'A', 'B'],
    'revenue': [200, 300, 150, 250']
})

pivot = df.pivot_table(
    values  = 'revenue',    # column to aggregate
    index   = 'region',     # rows
    columns = 'product',    # columns
    aggfunc = 'sum',        # aggregation (default: mean)
    fill_value = 0          # replace NaN
)
# product   A    B
# region
# North   200  300
# South   150  250
pivot — no aggregation (values must be unique per cell)
# Each (index, columns) pair must appear exactly once
df.pivot(index='region', columns='product', values='revenue')
melt — wide → long (unpivot)
wide = pd.DataFrame({
    'name': ['Alice', 'Bob'],
    'math': [90, 80],
    'eng':  [85, 75']
})

long = wide.melt(
    id_vars    = 'name',           # columns to keep as-is
    value_vars = ['math', 'eng'],  # columns to unpivot
    var_name   = 'subject',        # new column for former headers
    value_name = 'score'           # new column for values
)
# name subject  score
# Alice   math     90
# Alice    eng     85
# Bob     math     80
# Bob      eng     75
stack / unstack — MultiIndex ↔ columns
# stack  — move innermost column level → innermost row level
stacked = pivot.stack()       # columns become a new row-level

# unstack — move innermost row level → innermost column level
unstacked = stacked.unstack() # reverses stack
crosstab — frequency tables
# Count occurrences of (region, product) pairs
pd.crosstab(df['region'], df['product'])

# With custom aggregation
pd.crosstab(df['region'], df['product'],
            values=df['revenue'], aggfunc='sum')
Wide vs Long: Most Pandas operations (groupby, merge) prefer long format. Visualisation libraries like Seaborn also prefer long format. Use pivot_table / pivot for display or reporting; use melt to restore tidy/long format before analysis.

📅 Time Series Indexing

Pandas has first-class support for datetimes. Setting a DatetimeIndex unlocks powerful slicing, resampling, and rolling-window operations.

Creating DatetimeIndex
# Parse strings to datetime
dates = pd.to_datetime(['2024-01-01', '2024-01-02', '2024-01-03'])

# Generate a date range
idx = pd.date_range(start='2024-01-01', periods=365, freq='D')  # daily
idx = pd.date_range('2024-01-01', '2024-12-31', freq='ME')  # month-end
idx = pd.date_range('2024-01-01', periods=52,  freq='W')   # weekly

ts = pd.Series(np.random.randn(365), index=idx)

# Parse a column when reading CSV
df = pd.read_csv('sales.csv', parse_dates=['date'], index_col='date')
Common Frequency Aliases
'D'

Calendar day

'B'

Business day

'h' / 'min' / 's'

Hour / minute / second

'W'

Weekly (Sunday)

'ME' / 'MS'

Month-end / Month-start

'QE' / 'YE'

Quarter-end / Year-end

Date-based Slicing
# Partial-string indexing — Pandas resolves to full range automatically
ts['2024']                  # entire year 2024
ts['2024-03']               # entire March 2024
ts['2024-03-01':'2024-03-15']  # inclusive date range

# .loc also works with strings on a DatetimeIndex
df.loc['2024-Q1']            # Q1 2024

# Attribute extraction
df.index.year
df.index.month
df.index.day_of_week         # 0=Monday … 6=Sunday
df['date'].dt.hour           # .dt accessor for Series columns
resample() — group by time period
# Down-sample: daily → monthly sum
ts.resample('ME').sum()

# Multiple aggregations
df.resample('W').agg({'revenue': 'sum', 'units': 'mean'})

# Up-sample: daily → hourly, then forward-fill gaps
ts.resample('h').ffill()
rolling() — moving windows
# 7-day rolling mean
ts.rolling(window=7).mean()

# 30-day rolling std (min 15 observations required)
ts.rolling(30, min_periods=15).std()

# Expanding (cumulative) window
ts.expanding().mean()
shift() — lag / lead values
# Lag by 1 period (compare today vs yesterday)
ts.shift(1)

# Day-over-day percentage change
ts.pct_change()

# Period-over-period difference
ts.diff(1)
Time Zone Handling
# Localise naive datetime to a timezone
ts_utc = ts.tz_localize('UTC')

# Convert to a different timezone
ts_ist = ts_utc.tz_convert('Asia/Kolkata')

# Strip timezone info
ts_naive = ts_utc.tz_localize(None)
Remember: resample() requires a DatetimeIndex. If your dates are in a regular column, call df.set_index('date') first (or pass on='date' to resample). Also note that frequency strings changed in Pandas 2.2 — use 'ME' instead of the deprecated 'M'.

∅ Missing Data

Pandas uses NaN (from NumPy) as its sentinel for missing numeric values, and pd.NA for nullable integer/string dtypes. Handling NaNs correctly is one of the most common real-world tasks.

Detecting Missing Values
# Boolean mask — True where value is missing
df.isna()           # alias: df.isnull()
df.notna()          # alias: df.notnull()

# Count NaNs per column
df.isna().sum()

# Percentage missing
(df.isna().mean() * 100).round(2)

# Rows with ANY missing value
df[df.isna().any(axis=1)]
Dropping Missing Values
# Drop rows where ANY column is NaN
df.dropna()

# Drop rows only if ALL values are NaN
df.dropna(how='all')

# Drop rows missing values in specific columns
df.dropna(subset=['revenue', 'units'])

# Drop columns with more than 50% NaN
df.dropna(axis=1, thresh=int(len(df) * 0.5))
Filling Missing Values
# Fill all NaNs with a scalar
df.fillna(0)
df['score'].fillna(df['score'].mean())

# Fill per-column with a dict
df.fillna({'score': 0, 'grade': 'Unknown'})

# Forward-fill (propagate last valid value forward)
df.ffill()          # was .fillna(method='ffill') pre-2.0

# Backward-fill
df.bfill()

# Limit consecutive fills
df.ffill(limit=2)
Interpolation
# Linear interpolation between known values
ts.interpolate(method='linear')

# Time-aware interpolation (requires DatetimeIndex)
ts.interpolate(method='time')

# Polynomial / spline
ts.interpolate(method='polynomial', order=2)
Nullable Integer / String Dtypes
# Standard int64 cannot hold NaN — use nullable Int64
df['age'] = df['age'].astype('Int64')   # capital I
df['name'] = df['name'].astype('string')  # pd.StringDtype
Rule of thumb: Drop NaNs only when the row is genuinely unusable. For numeric columns prefer fillna(median) over fillna(mean) when outliers are present. For time-series, prefer ffill() / interpolate() over dropping.

Aa String Operations

The .str accessor vectorises Python string methods over a Series, avoiding slow Python loops. It also handles NaN gracefully — NaN in, NaN out.

Searching & Matching
s = df['name']

# Check if string contains a pattern (regex by default)
s.str.contains('ali', case=False, na=False)

# Starts / ends with
s.str.startswith('A')
s.str.endswith('e')

# Full regex match
s.str.match(r'^[A-Z]')         # anchored at start
s.str.fullmatch(r'[A-Za-z]+')  # entire string must match

# Extract capture groups into columns
s.str.extract(r'(\w+)\s(\w+)', expand=True)  # returns DataFrame
Cleaning & Transforming
# Case conversion
s.str.lower()  /  s.str.upper()  /  s.str.title()

# Strip whitespace (also lstrip / rstrip)
s.str.strip()

# Replace — plain string or regex
s.str.replace('  ', ' ', regex=False)     # collapse double spaces
s.str.replace(r'\d+', 'NUM', regex=True)  # replace runs of digits

# Zero-pad numbers stored as strings
s.str.zfill(5)   # '42' → '00042'

# Pad / centre
s.str.ljust(10, fillchar='.')
s.str.center(10)
Splitting
# Split on delimiter → list per cell
s.str.split(', ')

# Split and expand into separate columns
s.str.split(' ', n=1, expand=True)   # n=1 → max 1 split

# Split and explode into separate rows
df.assign(tag=df['tags'].str.split('|')).explode('tag')

# Partition on first occurrence of separator
s.str.partition('@')   # returns 3-column DataFrame
Useful Helpers
s.str.len()

Length of each string.

s.str.count(pat)

Count occurrences of pattern.

s.str.get(i)

Character at index i.

s.str.cat(sep=',')

Concatenate all strings into one.

s.str.slice(0, 3)

Substring via slice.

s.str.encode('utf-8')

Encode to bytes.

Performance note: .str methods are vectorised but still slower than NumPy numeric ops because strings are Python objects. For massive string columns (millions of rows), consider pd.StringDtype backed by Arrow ('string[pyarrow]') which is significantly faster.

⚡ Performance & Pitfalls

Knowing why something is slow — or silently wrong — is just as important as knowing what function to call.

Views vs Copies — the #1 gotcha
# Slicing often returns a VIEW (same memory), not a copy
subset = df[df['score'] > 80]   # may be a view or copy — undefined
subset['grade'] = 'A'           # ⚠ SettingWithCopyWarning!

# Safe pattern 1 — explicit copy
subset = df[df['score'] > 80].copy()
subset['grade'] = 'A'           # ✓ modifies only the copy

# Safe pattern 2 — use .loc on the original
df.loc[df['score'] > 80, 'grade'] = 'A'  # ✓ in-place on df

# Pandas 3.0+ Copy-on-Write (CoW) — opt-in now, default later
pd.options.mode.copy_on_write = True   # eliminates the ambiguity
.apply() is slow — prefer vectorised operations
# ❌ Slow — Python loop per row
df['tax'] = df['revenue'].apply(lambda x: x * 0.18)

# ✓ Fast — NumPy-backed vectorised operation
df['tax'] = df['revenue'] * 0.18

# ✓ Fast — use np.where instead of apply for conditions
df['label'] = np.where(df['score'] >= 90, 'A', 'B')

# ✓ Fast — pd.cut for binning
df['band'] = pd.cut(df['score'], bins=[0,60,80,100], labels=['C','B','A'])
Memory Optimisation
# Check memory usage
df.memory_usage(deep=True).sum() / 1024**2   # MB

# Downcast numeric columns
df['score'] = pd.to_numeric(df['score'], downcast='integer')
df['price'] = pd.to_numeric(df['price'], downcast='float')

# Convert low-cardinality string column to Categorical (~10x less memory)
df['region'] = df['region'].astype('category')

# Read only needed columns from CSV
df = pd.read_csv('big.csv', usecols=['date', 'revenue'])

# Read in chunks to process files larger than RAM
for chunk in pd.read_csv('huge.csv', chunksize=100_000):
    process(chunk)
eval() and query() for large DataFrames
# pd.eval — avoids creating large temporary arrays
df['result'] = pd.eval('df.a * df.b + df.c')

# df.query — readable filtering, faster on large frames
df.query('region == "North" and score > 80')

# Use @ to reference Python variables inside query strings
threshold = 80
df.query('score > @threshold')
Common Gotchas
  • Chained indexing: df['col'][0] = x may silently fail — always use df.loc[0, 'col'] = x.
  • inplace=True is not faster: It does not skip a copy internally; it just reassigns in-place. Often just a style choice.
  • Object dtype columns: A column with mixed types (int + str) becomes object dtype — much slower than homogeneous dtypes. Clean data on ingestion.
  • Iterating rows is slow: df.iterrows() is ~100× slower than vectorised ops. If you must iterate, use df.itertuples() or df.to_numpy().
  • Index alignment: Arithmetic on two Series automatically aligns on index labels before computing — unexpected if indices differ.
Profile before optimising: Use %timeit in Jupyter or df.memory_usage(deep=True) to find the real bottleneck before rewriting code. The biggest wins usually come from fixing dtypes and eliminating .apply() calls.

? FAQ — Common Questions

Questions that came up while learning Pandas, with clear answers.

How does mean() work?

It computes the arithmetic mean (sum ÷ count). The key is the axis parameter:

# On a Series
s = pd.Series([10, 20, 30, 40])
s.mean()          # → 25.0

# axis=0 (default) → mean of each COLUMN
df.mean()

# axis=1 → mean of each ROW
df.mean(axis=1)

# NaN is skipped by default
pd.Series([10, float('nan'), 30]).mean()              # → 20.0
pd.Series([10, float('nan'), 30]).mean(skipna=False)  # → NaN
What does "normalise" mean?

Rescaling values to a common scale so they're comparable, removing the effect of units or magnitude differences. Two common types:

Min-Max → [0, 1]

(x - min) / (max - min)
Smallest → 0, Largest → 1

Z-score → mean=0, std=1

(x - mean) / std
Below average → negative, above → positive

# Min-Max
df['norm'] = (df['score'] - df['score'].min()) / (df['score'].max() - df['score'].min())

# Z-score
df['z'] = (df['score'] - df['score'].mean()) / df['score'].std()
What type of operation is groupby + transform for percentages?

It's a group-wise normalisation — each value is expressed as a fraction of its group's total. The result type is a pd.Series of floats, same shape as the original.

# Each row = its share of the region's total revenue
sales['rev_pct'] = (
    sales['revenue'] /
    sales.groupby('region')['revenue'].transform('sum')
)
# transform() keeps the same shape as the original DataFrame
# so the division works row-by-row perfectly
# values within each group sum to 1.0
What is standard deviation?

A measure of how spread out values are from the mean. Small std → values cluster near the mean. Large std → values are widely scattered.

# How it's calculated (manually)
# scores = [60, 70, 80, 90, 100], mean = 80
# Step 1: distances from mean → [-20, -10, 0, +10, +20]
# Step 2: square them         → [400, 100, 0, 100, 400]
# Step 3: average (variance)  → 200
# Step 4: square root (std)   → ≈ 14.14

s = pd.Series([60, 70, 80, 90, 100])
s.mean()   # → 80.0
s.std()    # → 15.81  (Pandas uses n-1 by default)
s.var()    # → 250.0  (variance = std²)
s.std(ddof=0)  # → 14.14  (divide by n, not n-1)
68-95-99.7 rule (for normal distributions): mean ± 1 std covers ~68% of values · ± 2 std covers ~95% · ± 3 std covers ~99.7%
Difference between RIGHT and OUTER join?
RIGHT join

All rows from the right table. Left-unmatched rows are dropped. NaN where right has no left match.

OUTER (full) join

All rows from both tables. Nothing is ever dropped. NaN wherever there's no match on either side.

# RIGHT — Carol (dept 99) gets dropped; Design (no employees) kept with NaN
pd.merge(employees, departments, on='dept_id', how='right')

# OUTER — Carol AND Design both kept; NaN where no match
pd.merge(employees, departments, on='dept_id', how='outer')
One-liner: RIGHT is a one-sided guarantee (right table always wins). OUTER is a both-sided guarantee — nothing gets dropped, ever.
What's the difference between pd.concat axis=0 and axis=1?
# axis=0 (default) — grows DOWNWARD, adds more rows
pd.concat([df1, df2], ignore_index=True)

# axis=1 — grows RIGHTWARD, adds more columns
pd.concat([df1, df2], axis=1)

# Visual:
# axis=0:   axis=1:
# ┌─────┐   ┌────┬────┐
# │ df1 │   │df1 │df2 │
# ├─────┤   └────┴────┘
# │ df2 │
# └─────┘

ignore_index=True only resets the row index to 0,1,2… — it has nothing to do with column alignment.

Why does pd.concat produce NaN values?

Because the two DataFrames don't share the same column names. When concat stacks rows (axis=0), it builds a union of all columns — where a DataFrame is missing a column, it fills with NaN.

a = pd.DataFrame({'A': [1, 2]})  # only has column A
b = pd.DataFrame({'B': [3, 4]})  # only has column B

pd.concat([a, b], ignore_index=True)
#      A    B
# 0  1.0  NaN   ← from a (no B column)
# 1  2.0  NaN   ← from a (no B column)
# 2  NaN  3.0   ← from b (no A column)
# 3  NaN  4.0   ← from b (no A column)

# FIX: if they are meant side-by-side, use axis=1
pd.concat([a, b], axis=1)
#    A  B
# 0  1  3
# 1  2  4
